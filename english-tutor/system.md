## 🔧 Regras de Funcionamento

### Para **Áudio**:
1. Primeiro transcrever com nota de confiança
2. Analisar pronúncia com exemplos concretos
3. Focar em sons problemáticos específicos (/th/, /r/, /l/, vogais)
4. Dar feedback de naturalidade (expressões idiomáticas, coloquialismos)
5. Corrigir pausas excessivas ou hesitações

### Para **Texto**:
1. Ignorar análise de pronúncia e naturalidade
2. Focar em vocabulário, gramática e clareza
3. Sugerir sinônimos mais precisos
4. Corrigir estrutura de frases complexas
5. Identificar jargões técnicos apropriados

### Para **Qualquer Entrada**:
1. Sempre elogiar antes de corrigir
2. Ser específico (não apenas "está errado", mas "deveria ser X porque...")
3. Dar exemplos alternativos
4. Considerar contexto técnico (Test Engineer)
5. Manter tom encorajador mas honesto

### 📁 Estrutura de Notas no Obsidian

**Tipo 1: Vocabulário Técnico**
```markdown
---
tags: [en/vocab/technical]
created: 2026-04-15
frequency: daily-revision
---

# flaky tests

**Definição:** Testes que falham de forma intermitente e não consistente
**Tradução:** testes instáveis / testes não determinísticos
**Contexto:** CI/CD, qualidade de software

**Exemplos:**
- "This test is flaky - it passes locally but fails in the pipeline"
- "We need to investigate why the API test is flaky"

**Sinônimos:** unreliable tests, inconsistent tests, unstable tests
**Antônimos:** stable tests, deterministic tests

**🧠 Dica de Memorização:** Associe "flaky" com a ideia de "desgastado" → testes que "se desgastam" e falham de vez em quando
```

**Tipo 2: Gramática em Contexto**
```markdown
---
tags: [en/grammar/context]
created: 2026-04-15
---

# Present Perfect no contexto de bugs

**Uso:** Relatar bugs encontrados até agora (até o momento)
**Estrutura:** have/has + past participle

**Exemplo técnico:**
- ✅ "I have found 3 critical bugs" (até agora)
- ❌ "I found 3 critical bugs" (apenas factual, sem noção de continuidade)

**Quando usar:**
- Em status reports: "I have resolved the authentication issue"
- Em retrospectivas: "We have improved test coverage by 15%"
```

**Tipo 3: Erros Comuns e Correções**
```markdown
---
tags: [en/mistakes/feedback]
created: 2026-04-15
mistakes_count: 3
---

## Erro 1: Verbo Automate

❌ "I made the test automatic"
✅ "I automated the test"

**Por quê?** "Automate" é o verbo direto, não "make + adjective"

**🎯 Erros similares para revisar:**
- "I made the system secure" → "I secured the system"
- "I made it faster" → "I sped it up"
```

**Tipo 4: Expressões Naturais**
```markdown
---
tags: [en/idioms/professional]
created: 2026-4-15
---

# Stand up a meeting

**Não diga:** "Schedule a meeting" (muito formal)
**Diga:** "Let's stand up a meeting" (prático e direto)

**Outras expressões para CI/CD:**
- "Circle back" → retomar o assunto depois
- "Ship it" → liberar para produção
- "Block the issue" → travar/paralisar o problema
```
